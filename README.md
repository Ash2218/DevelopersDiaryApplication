# Developers Diary

Developers Diary is a Windows desktop application that helps developers build a personal programming knowledge base by recording errors, solutions, 
explanations, and reusable code snippets. Instead of repeatedly searching online or asking AI the same questions, developers can store their debugging 
experience in one searchable location and build long-term understanding.


## The Problem

Developers frequently encounter the same programming errors across different projects but often lose the knowledge gained while solving them.

Common challenges include:

- Repeatedly searching for the same solutions.
- Forgetting why previous solutions worked.
- Keeping notes across multiple documents and applications.
- Limited space within source code for detailed learning notes.
- Over-reliance on AI tools instead of building long-term understanding.



## The Solution

Developers Diary provides a searchable personal knowledge base where developers can:

- Record programming errors.
- Save one or more solutions.
- Generate AI explanations for technical error messages.
- Write personal explanations in their own words.
- Store reusable code snippets.
- Organise projects by programming language.
- Search previous debugging experiences and code snippets.



## Features

- User authentication
- Project management
- Programming language management
- Error recording
- Multiple solutions per error
- AI-generated error explanations
- Personal explanations
- Code snippet library
- Categories
- Search and filtering
- SQL Server database integration



## Motivation

Modern AI tools make solving programming problems faster but often encourage developers to repeatedly ask the same questions 
instead of learning from previous experiences.

As a Computer Science student, I found myself encountering the same errors across different projects and storing notes in 
multiple places. I wanted to create a tool that allows developers to organise their knowledge, revisit previous solutions, 
and better understand the reasoning behind their code rather than relying solely on AI conversations.
